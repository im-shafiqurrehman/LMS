# Rate Limiting

Rate limiting prevents brute-force attacks on the auth endpoints. A simple time-window rate limiter:

```js
import rateLimit from 'express-rate-limit';

export const authRateLimit = rateLimit({
  windowMs:         15 * 60 * 1000,  // 15 minutes
  max:              10,               // 10 requests per window per IP
  message:          { error: 'Too many requests. Please try again later.' },
  standardHeaders:  true,
  legacyHeaders:    false,
});
```

Install: npm install express-rate-limit

Apply only to auth routes (not the entire API):
```js
authRouter.post('/login',    authRateLimit, validate(loginSchema),    loginHandler);
authRouter.post('/register', authRateLimit, validate(registerSchema), registerHandler);
```

For production, use Redis as the rate limit store (express-rate-limit has a Redis store adapter). In-memory rate limiting does not work across multiple server instances.
