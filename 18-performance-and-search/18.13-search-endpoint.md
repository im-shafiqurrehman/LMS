# Full-Text Search Endpoint

Update the listCourses service to use Atlas Search when a query parameter is present:

```js
async function listCourses({ q, categoryId, cursor, limit = 12 }) {
  if (q) {
    // Use Atlas Search for keyword queries
    const courses = await Course.aggregate([
      { $search: { index: 'default', text: { query: q, path: ['title', 'description'], fuzzy: {} } } },
      { $match: { status: 'published', ...(categoryId ? { categoryId } : {}) } },
      { $limit: limit },
      { $lookup: { from: 'users', localField: 'instructorId', foreignField: '_id', as: 'instructor', pipeline: [{ $project: { email: 1 } }] } },
      { $unwind: '$instructor' },
    ]);
    return { courses, hasMore: false, nextCursor: null };
  }
  // Use standard MongoDB query with cursor pagination for non-search requests
  // ...
}
```

Atlas Search does not support cursor-based pagination the same way — return all matching results up to the limit.
