# Cache Invalidation

Cache invalidation is famously one of the two hard things in computer science (along with naming things and off-by-one errors).

For the course catalogue:

```js
async function invalidateCatalogueCache() {
  const keys = await redis.keys('courses:list:*');
  if (keys.length > 0) await redis.del(keys);
}
```

Call this in the service after:
- Admin approves a course (status → published)
- Admin rejects a course (status → rejected)
- Instructor updates a published course (title, description, price — if this is allowed)

In Redis, keys('courses:list:*') scans all keys matching the pattern. For a large Redis instance with millions of keys, use SCAN instead of KEYS (KEYS is O(N) and blocks other commands). For EduFlow's scale, KEYS is fine.
