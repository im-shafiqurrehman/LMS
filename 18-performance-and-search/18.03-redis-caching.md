# Redis Caching Strategy

Caching the catalogue makes sense because:
- The catalogue is read far more often than it is written
- The data changes when a course is published or updated (relatively rare)
- A cache miss falls back to MongoDB seamlessly

Caching strategy:
- Cache key: courses:list:{queryHash} — includes the filter and cursor parameters
- TTL: 300 seconds (5 minutes)
- Invalidation: delete all courses:list:* keys when a course is published or updated

> Worth reading: search "Redis caching patterns cache aside invalidation" — the cache-aside pattern is what EduFlow uses.
