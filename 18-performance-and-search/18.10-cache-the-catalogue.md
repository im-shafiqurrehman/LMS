# Cache the Course Catalogue

Update the listCourses service function to check Redis before querying MongoDB:

```js
import redis from '../../lib/redis.js';

async function listCourses(params) {
  const cacheKey = `courses:list:${JSON.stringify(params)}`;

  // 1. Check cache
  const cached = await redis.get(cacheKey);
  if (cached) return JSON.parse(cached);

  // 2. Cache miss — query MongoDB
  const result = await queryCoursesFromMongoDB(params);

  // 3. Store in cache with 5-minute TTL
  await redis.setex(cacheKey, 300, JSON.stringify(result));

  return result;
}
```

The first request populates the cache. Subsequent identical requests (same filter, same cursor) return the cached JSON without touching MongoDB.

✅ Verify: make the same GET /api/courses request twice; check MongoDB logs — the second request should not trigger a MongoDB query.
