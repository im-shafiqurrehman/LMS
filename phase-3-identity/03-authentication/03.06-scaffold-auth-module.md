# Scaffold the Auth Module

The schema is done. Now create the files that implement authentication. You scaffold first — create the structure, then fill it in step by step. This way you always know where things live before you write the logic.

## Where the project is now

`src/modules/` exists but is empty. By the end of this sub-chapter, the `auth/` module folder exists with all the files it needs — empty, but in place.

## Create the folder and files

```bash
mkdir -p src/modules/auth
touch src/modules/auth/auth.routes.js
touch src/modules/auth/auth.service.js
touch src/modules/auth/auth.validators.js
touch src/modules/auth/auth.helpers.js
```

**What each file does:**

- `auth.routes.js` — defines the HTTP routes (`POST /auth/register`, `POST /auth/login`, etc.) and wires them to service functions. No business logic here.
- `auth.service.js` — the business logic: hash passwords, query the database, issue tokens, send emails. This is where the real work happens.
- `auth.validators.js` — Zod schemas that validate the shape and content of incoming request bodies before they touch the service layer.
- `auth.helpers.js` — small, reusable utilities: generating a token pair, hashing a refresh token, generating an OTP.

## Wire the router into the app

Open `src/app.js` and add the auth router:

```js
// In app.js — import and mount the auth router
// Mount it at /api/auth so routes become /api/auth/register, /api/auth/login, etc.
```

The auth routes don't exist yet — your import will fail until you export a router from `auth.routes.js`. Add an empty Express router export to `auth.routes.js` now so the app boots:

```js
// auth.routes.js — minimum to make the app boot
import { Router } from 'express';
export const authRouter = Router();
// routes added in the next sub-chapters
```

Run `npm run dev` and confirm the server still starts without errors. You have not broken anything; you've created the space to build in.

## The routes you will add (overview)

| Method | Path | What it does |
|---|---|---|
| `POST` | `/api/auth/register` | Creates an account, sends verification OTP |
| `POST` | `/api/auth/verify-email` | Confirms the OTP, marks account verified |
| `POST` | `/api/auth/login` | Validates credentials, returns token pair |
| `POST` | `/api/auth/refresh` | Accepts a refresh token, returns new access token |
| `POST` | `/api/auth/logout` | Revokes the refresh token |
| `POST` | `/api/auth/forgot-password` | Sends a password reset OTP |
| `POST` | `/api/auth/reset-password` | Accepts OTP + new password, updates hash |

You will add them one at a time across the next sub-chapters, verifying each before moving to the next.

❌ Don't add all seven routes at once before any of them work — add, verify, continue.
