# What a JWT Is

A JSON Web Token (JWT) is a compact, URL-safe string that encodes a set of claims — facts about a user — and proves those claims have not been tampered with since the server produced them.

## The three parts

A JWT looks like this:

```
eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJzdWIiOiJ1c2VyMTIzIiwicm9sZSI6InN0dWRlbnQiLCJpYXQiOjE2OTUwMDAwMDAsImV4cCI6MTY5NTAwMDkwMH0.SflKxwRJSMeKKF2QT4fwpMeJf36POk6yJV_adQssw5c
```

It is three Base64-encoded strings separated by dots:

1. **Header** — the algorithm used to sign the token (`HS256` for HMAC-SHA256) and the token type (`JWT`).
2. **Payload** — the claims: `sub` (subject — the user's ID), `role`, `iat` (issued at), `exp` (expiry timestamp).
3. **Signature** — `HMAC-SHA256(base64(header) + "." + base64(payload), secretKey)`.

The signature is what matters. It is computed using a secret key that only the server knows. If anyone changes the payload (for example, to elevate their role from `student` to `admin`), the signature no longer matches. The server detects the mismatch when it verifies the token and rejects the request.

## What JWTs are not

JWTs are **not encrypted**. The payload is Base64-encoded, which is trivially reversible — anyone who holds the token can read its contents. Never put sensitive data (passwords, payment info, private notes) in a JWT payload. For EduFlow, the payload contains only the user's ID, role, and expiry — nothing sensitive.

JWTs are also not a session. The server does not store them. This is their strength (stateless, scalable) and their weakness (cannot be individually revoked before expiry). The two-token pattern in the previous sub-chapter handles that weakness.

## Signing and verifying in Node.js

The `jsonwebtoken` library handles both:

```js
import jwt from 'jsonwebtoken';

// Issue a token
const accessToken = jwt.sign(
  { sub: user._id.toString(), role: user.role },
  config.jwtSecret,
  { expiresIn: config.jwtAccessExpiry }  // "15m"
);

// Verify a token (throws if invalid or expired)
const payload = jwt.verify(token, config.jwtSecret);
// payload.sub → user ID; payload.role → "student" | "instructor" | "admin"
```

You will use this exact pattern in the register, login, and refresh endpoints.

Install the library now:

```
npm install jsonwebtoken
npm install --save-dev @types/jsonwebtoken
```
