# Ownership Enforcement — The Pattern

Every write route in this chapter uses the ownership check pattern from chapter 08. Here is the exact pattern to follow for every service function:

```js
async function updateCourse(courseId, userId, role, updates) {
  // 1. Find the resource
  const course = await Course.findById(courseId);
  if (!course) throw new AppError('Course not found', 404);

  // 2. Check ownership (admin bypass where appropriate)
  if (role !== 'admin' && course.instructorId.toString() !== userId) {
    throw new AppError('Forbidden', 403);
  }

  // 3. Business rule: cannot edit a published or pending course
  if (['published', 'pending'].includes(course.status)) {
    throw new AppError('Cannot edit a published or pending course', 422);
  }

  // 4. Perform the operation
  Object.assign(course, updates);
  return course.save();
}
```

This pattern — find, check ownership, check business rules, act — appears in every write function. Do not shortcut it.
