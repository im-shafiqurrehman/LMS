# Submit for Review

PATCH /api/instructor/courses/:id/submit — changes status from 'draft' to 'pending'.

```js
async function submitForReview(courseId, userId) {
  const course = await Course.findById(courseId);
  if (!course) throw new AppError('Course not found', 404);
  if (course.instructorId.toString() !== userId) throw new AppError('Forbidden', 403);
  if (course.status !== 'draft') throw new AppError('Course is not in draft status', 422);
  if (course.totalLessons === 0) throw new AppError('Add at least one lesson before submitting', 422);

  course.status = 'pending';
  return course.save();
}
```

Once submitted:
- The instructor cannot edit the course (the update/addLesson endpoints check status !== 'draft')
- The admin sees it in the pending queue (chapter 19)
- The admin can approve (status → published) or reject (status → rejected with feedback)

✅ Submit changes status to 'pending'.
✅ Submitting a course with 0 lessons returns 422.
✅ Submitting an already-pending course returns 422.
