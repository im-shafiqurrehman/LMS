# What You Will Build

Backend endpoints for instructor course management:

| Method | Path | Who | What |
|--------|------|-----|------|
| POST | /api/courses | instructor | Create a draft course |
| GET | /api/courses/mine | instructor | List my courses |
| PATCH | /api/courses/:id | instructor | Update course metadata |
| DELETE | /api/courses/:id | instructor | Delete a draft course |
| POST | /api/courses/:id/lessons | instructor | Add a lesson |
| PATCH | /api/courses/:id/lessons/:lid | instructor | Update a lesson |
| DELETE | /api/courses/:id/lessons/:lid | instructor | Delete a lesson |
| PATCH | /api/courses/:id/submit | instructor | Submit for review |
| POST | /api/courses/:id/cover | instructor | Upload cover image |

All routes require authenticate + authorize('instructor', 'admin').
All write routes check ownership: course.instructorId must match req.user.id (unless admin).
