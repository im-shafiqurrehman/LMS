# The Route Table

Before writing the routes, sketch the module structure:

```
server/src/modules/courses/
├── courses.routes.js
├── courses.service.js
└── courses.validators.js
```

Mount in app.js:
```js
import { coursesRouter } from './modules/courses/courses.routes.js';
app.use('/api/courses', coursesRouter);
```

Important: both the catalogue router and the courses router are mounted at /api/courses. Express resolves them in order — the catalogue router handles public GET routes, the courses router handles authenticated POST/PATCH/DELETE routes. Mount the catalogue router first so it handles GET requests before the courses router.

Actually, to avoid conflicts, rename the instructor routes module to /api/instructor and the admin routes to /api/admin. This is cleaner:
- GET /api/courses → public catalogue
- POST /api/instructor/courses → instructor creates a course
- GET /api/instructor/courses/mine → instructor lists their courses

Update app.js accordingly.
