# Keyword Search with Regex

For basic keyword search (before adding Atlas Search in chapter 18), use MongoDB regex queries:

```js
if (q) {
  filter.$or = [
    { title:       { $regex: q, $options: 'i' } },
    { description: { $regex: q, $options: 'i' } },
  ];
}
```

**The caveat:** regex queries on unindexed fields cause a full collection scan. This is acceptable for development and early production. When the course collection grows, you will add a text index or switch to Atlas Search (chapter 18).

**Add a text index** now to prepare:

```js
// In course.model.js
courseSchema.index({ title: 'text', description: 'text' });
```

A text index enables the $text query operator — more efficient than $regex for keyword matching:

```js
if (q) filter.$text = { $search: q };
```

Commit both approaches — regex as a fallback comment, $text as the implementation.

✅ GET /api/courses?q=react returns courses matching "react" in title or description.
✅ The query uses the text index (verify with explain()).
