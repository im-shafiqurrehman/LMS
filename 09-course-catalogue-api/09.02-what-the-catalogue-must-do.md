# What the Catalogue Must Do

The catalogue has three query patterns that drive the implementation:

**1. Browse all published courses** (most common)
GET /api/courses?status=published&limit=12&cursor=<lastId>
Returns: paginated list of courses with instructor name

**2. Filter by category**
GET /api/courses?categoryId=<id>&status=published&limit=12
Returns: same paginated list, filtered to one category

**3. Keyword search**
GET /api/courses?q=javascript&status=published
Returns: courses whose title or description matches the keyword

**Plus two detail queries:**

**4. Single course detail**
GET /api/courses/:id
Returns: course + full lesson list (titles and durations, not video URLs) + instructor info

**5. Course lessons**
GET /api/courses/:id/lessons
Returns: lesson list sorted by position (used by the lesson viewer)

Everything in this chapter is public — no authentication required. Any visitor can browse the catalogue without logging in. Lesson video URLs are not included in the catalogue — only in the lesson access API (chapter 15), which requires enrollment.
