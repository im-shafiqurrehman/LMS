# Recap and What Is Next

This chapter built the read-heavy catalogue API:

- GET /api/courses — paginated list with category filter and keyword search
- GET /api/courses/:id — single course detail with lessons (no video URLs)
- GET /api/courses/:id/lessons — ordered lesson list for the lesson player
- Cursor pagination (not offset)
- N+1 prevention via .populate()
- Request parameter validation with Zod
- Indexes verified and query plans confirmed

What is next: Chapter 10 — Browse Courses (Frontend). The Next.js course catalogue page consumes the API you just built — filters, pagination, course cards, and the course detail page.
