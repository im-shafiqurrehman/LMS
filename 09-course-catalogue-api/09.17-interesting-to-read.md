# Interesting to Read After This Chapter

These are not required reading — but if you are curious about the production considerations behind the decisions in this chapter, these are worth your time.

**1. Cursor vs offset pagination at scale**
Search "cursor pagination vs offset pagination MongoDB performance" — there are excellent blog posts from companies that migrated from offset to cursor and the latency improvements they saw.

**2. MongoDB Explain Plans**
Search "MongoDB explain executionStats IXSCAN COLLSCAN". Understanding explain output is the most important performance debugging skill you will use throughout your career.

**3. MongoDB Atlas Search vs regex search**
Search "MongoDB Atlas Search full-text search vs regex". The regex approach in this chapter works well at small scale. Atlas Search (chapter 18) replaces it with a proper full-text search index — understanding why the switch is worth it prepares you for that chapter.
