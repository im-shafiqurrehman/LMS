# Course Catalogue API — Set the Scene

The catalogue is the front door of EduFlow. Every visit by a prospective student goes through it. It has to be fast, filterable, and paginated. Getting it right matters — a catalogue that returns 500 courses in one response will bring the server to its knees at scale.

This chapter builds the read-heavy catalogue API:
- List published courses with optional category and keyword filters
- Sort by newest / most popular
- Cursor-based pagination (more scalable than offset for large result sets)
- Single course detail with lesson preview

By the end, the catalogue is queryable with curl and ready for the Next.js frontend in chapter 10.

Prerequisites:
- Chapter 05 seed data (published courses exist)
- Chapter 06 authentication (protected routes later in the chapter)
- Chapter 08 authorization (the catalogue is public — no auth needed for reading)
