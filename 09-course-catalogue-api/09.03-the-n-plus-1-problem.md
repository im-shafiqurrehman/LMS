# The N+1 Query Problem

The catalogue needs to return each course with its instructor's name and email. The wrong way to do this:

```js
// WRONG — N+1 queries
const courses = await Course.find({ status: 'published' });
for (const course of courses) {
  course.instructor = await User.findById(course.instructorId); // one query per course!
}
```

For 12 courses, this issues 13 queries: 1 for the courses + 12 for the instructors. For 100 courses, 101 queries. This is the N+1 problem.

**The fix: use .populate() or a $lookup aggregation**

```js
// CORRECT — 2 queries total (courses + all referenced users in one go)
const courses = await Course.find({ status: 'published' })
  .populate('instructorId', 'email')  // Mongoose resolves all instructorIds in one query
  .limit(12);
```

Mongoose's populate issues exactly 2 queries: one for the courses, one for all unique instructorId values found. Not one per course.

The alternative is MongoDB's $lookup aggregation pipeline — more powerful but more verbose. For EduFlow's catalogue queries, populate is sufficient and more readable.

> Worth reading: search "MongoDB N+1 query problem Node.js Mongoose" — the Mongoose documentation explains when to use populate vs aggregation.
