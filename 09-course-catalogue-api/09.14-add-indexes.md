# Verify Indexes Are in Place

Before testing the catalogue at any meaningful scale, verify the indexes exist in the running database.

```js
// Run in mongosh
use eduflow
db.courses.getIndexes()
```

You should see indexes for:
- { status: 1, createdAt: -1 }
- { categoryId: 1, status: 1 }
- { instructorId: 1 }
- { title: "text", description: "text" }

If any are missing, it means the schema file was written after the collection was first accessed — Mongoose creates indexes when the model is first loaded and the collection does not yet exist. Fix: drop and re-seed, or use db.courses.createIndex() directly.

**Verify query uses the index:**

```js
db.courses.find({ status: 'published' }).sort({ _id: -1 }).explain('executionStats')
```

Look for: stage: "IXSCAN" in the winning plan. If you see stage: "COLLSCAN", the index is not being used.

✅ IXSCAN for the catalogue query.
❌ COLLSCAN on a published catalogue query is a bug — fix before moving to the frontend.
