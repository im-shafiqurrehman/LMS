# Implement Cursor Pagination

Complete the listCourses service function with cursor pagination:

```js
// catalogue.service.js — listCourses implementation
async listCourses({ categoryId, q, cursor, limit = 12 }) {
  const filter = { status: 'published' };
  if (categoryId) filter.categoryId = categoryId;
  if (cursor)     filter._id = { $lt: new mongoose.Types.ObjectId(cursor) };
  if (q)          filter.$text = { $search: q };

  const courses = await Course
    .find(filter)
    .populate('instructorId', 'email')
    .populate('categoryId', 'name slug')
    .sort({ _id: -1 })
    .limit(limit + 1);

  const hasMore    = courses.length > limit;
  const items      = courses.slice(0, limit);
  const nextCursor = hasMore ? items[items.length - 1]._id : null;

  return { courses: items, nextCursor, hasMore };
}
```

The client uses nextCursor like this:
```
GET /api/courses               → { courses: [...], nextCursor: "abc123", hasMore: true }
GET /api/courses?cursor=abc123 → { courses: [...], nextCursor: "def456", hasMore: true }
GET /api/courses?cursor=def456 → { courses: [...], nextCursor: null, hasMore: false }
```

✅ Requesting with a cursor from the previous response returns the next page.
✅ When hasMore is false, the last page has been reached.
