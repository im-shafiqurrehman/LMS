# Filter by Category

Extend the listCourses service function to accept an optional categoryId query parameter:

```js
// In catalogue.service.js — within listCourses
const filter = { status: 'published' };

if (categoryId) filter.categoryId = categoryId;
if (cursor)     filter._id = { $lt: new mongoose.Types.ObjectId(cursor) };
if (q)          filter.$or = [
  { title:       { $regex: q, $options: 'i' } },
  { description: { $regex: q, $options: 'i' } },
];
```

The category filter is a simple equality match. With the compound index { categoryId, status } on the Course collection, MongoDB uses the index to find matching courses efficiently.

**Validate the query parameters** before they reach the service:

```js
// catalogue.validators.js
const listCoursesQuery = z.object({
  categoryId: z.string().optional(),
  q:          z.string().optional(),
  cursor:     z.string().optional(),
  limit:      z.coerce.number().min(1).max(50).default(12),
});
```

Parse query params in the route handler:
```js
const params = listCoursesQuery.parse(req.query);
const result = await catalogueService.listCourses(params);
```

✅ GET /api/courses?categoryId=<id> returns only courses in that category.
✅ GET /api/courses?q=javascript returns courses with "javascript" in the title.
