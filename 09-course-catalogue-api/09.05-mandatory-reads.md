# Mandatory Reads — Before the Catalogue

Before building the catalogue API, read two things:

**1. Mongoose Query API reference**
Search "Mongoose Query find populate sort limit". Understand how chaining works, when queries are actually executed (they are lazy — not until .exec() or await), and what populate does at the driver level.

**2. MongoDB indexes for read-heavy queries**
Search "MongoDB compound index query optimisation". Understand why the order of fields in a compound index matters for query planning. The catalogue uses an index on { status, createdAt } — understand why both fields must be in the index for the query to use it efficiently.

These reads take about 20 minutes and will save you significant debugging time when the catalogue query returns results in the wrong order or ignores the index.
