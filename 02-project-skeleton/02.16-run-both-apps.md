# Run Both Apps

You need two terminal windows — one for the backend, one for the frontend.

**Terminal 1 — Backend:**

```
cd server
npm run dev
```

Expected output:
```
EduFlow server running on port 3000
```

**Terminal 2 — Frontend:**

```
cd client
npm run dev -- --port 3001
```

Expected output:
```
▲ Next.js 14.x.x
- Local: http://localhost:3001
```

**Open http://localhost:3001 in your browser.** You should see the default Next.js homepage.

**Verify the health check from the frontend** — edit app/page.tsx to fetch from the backend and display the status:

```tsx
'use client';
import { useEffect, useState } from 'react';

export default function HomePage() {
  const [status, setStatus] = useState<string | null>(null);

  useEffect(() => {
    fetch(`${process.env.NEXT_PUBLIC_API_URL}/api/health`)
      .then(r => r.json())
      .then(d => setStatus(d.status));
  }, []);

  return <main><h1>EduFlow</h1><p>API status: {status ?? 'loading...'}</p></main>;
}
```

✅ You see "API status: ok" on the homepage — confirming the frontend reaches the backend.
❌ If you see a CORS error in the browser console, confirm FRONTEND_URL in server/.env matches http://localhost:3001.
