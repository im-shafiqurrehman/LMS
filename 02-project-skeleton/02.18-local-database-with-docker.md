# Run the Local Database with Docker

You need MongoDB and Redis running locally. Docker keeps them isolated from your machine and makes the setup reproducible.

Create docker-compose.yml in the project root:

```yaml
services:
  mongodb:
    image: mongo:7
    environment:
      MONGO_INITDB_ROOT_USERNAME: admin
      MONGO_INITDB_ROOT_PASSWORD: yourpassword
      MONGO_INITDB_DATABASE: eduflow
    ports:
      - "27017:27017"
    volumes:
      - mongodb_data:/data/db

  redis:
    image: redis:7
    ports:
      - "6379:6379"

volumes:
  mongodb_data:
```

**Start the services:**

```
docker compose up -d
```

**Verify MongoDB is reachable:**

```
docker compose exec mongodb mongosh -u admin -p yourpassword --eval "db.adminCommand('ping')"
```

You should see: { ok: 1 }

**Update server/.env with the correct DATABASE_URL:**

```
DATABASE_URL=mongodb://admin:yourpassword@localhost:27017/eduflow?authSource=admin
```

> Worth reading: search "Docker Compose for Node.js local development" — the official Docker documentation explains volumes (your data survives container restarts) and the difference between docker compose up and docker compose start.

✅ docker compose ps shows both mongodb and redis as running.
✅ Mongoose will connect using the DATABASE_URL in chapter 04.
❌ Do not use a cloud MongoDB instance for local development — your internet connection should not be a dependency on your build loop.
