# What You Will Build in This Chapter

Two apps, one repo. Here is the target state:

```
eduflow/
├── server/                   ← Express.js backend
│   ├── src/
│   │   ├── config/index.js   ← env validation
│   │   ├── lib/db.js         ← Mongoose connection
│   │   ├── middleware/
│   │   ├── modules/
│   │   ├── app.js
│   │   └── server.js
│   ├── .env
│   ├── .gitignore
│   └── package.json
├── client/                   ← Next.js frontend
│   ├── app/
│   ├── components/
│   ├── lib/api.ts
│   ├── store/
│   ├── types/index.ts
│   ├── .env.local
│   └── package.json
├── docker-compose.yml        ← local MongoDB + Redis
└── .gitignore
```

Specifically, after this chapter:
- GET /api/health returns { "status": "ok", "timestamp": "..." }
- The Next.js homepage fetches from the health endpoint and shows the status
- No secrets are in git log
- Both apps restart cleanly after a file change

The rest of the course builds features on top of this foundation.
