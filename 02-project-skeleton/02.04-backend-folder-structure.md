# Backend Folder Structure

Open your terminal in the project root. Create the server/ directory and its skeleton:

```
mkdir -p server/src/modules
mkdir -p server/src/models
mkdir -p server/src/lib
mkdir -p server/src/middleware
mkdir -p server/src/config
touch server/src/app.js
touch server/src/server.js
touch server/src/config/index.js
touch server/src/lib/db.js
touch server/src/middleware/errorHandler.js
```

**What each folder holds:**

| Folder / File | Purpose |
|---------------|---------|
| src/config/index.js | Read and validate all env variables at startup. One place for all config. |
| src/lib/db.js | Mongoose connection. Called once at boot. |
| src/middleware/ | authenticate, authorize, errorHandler — shared across all modules |
| src/models/ | Mongoose model files. Shared by all modules. |
| src/modules/ | One folder per feature (auth, courses, enrollments). Each holds its routes, service, and validators. |
| src/app.js | Builds the Express app: apply middleware, mount routes, export. |
| src/server.js | Entry point: connect DB, start listening. |

**Why module-based, not layer-based?**

Layer-based puts all routes together, all services together, all models together. Working on authentication means jumping between three folders. Module-based keeps everything for one feature in one place. When you add notifications, you add a notifications/ module — nothing else moves.

✅ Verify with: find server/src -type f — you should see all the files you created.
