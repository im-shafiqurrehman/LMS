# The Express Entry Point

Two files: app.js builds the application, server.js starts it. The split matters for testing — you can import app.js in tests without actually starting a server.

**src/app.js** — build and configure the app:

```js
// src/app.js — shape only
import express from 'express';
import cors    from 'cors';
import helmet  from 'helmet';
import morgan  from 'morgan';

const app = express();

app.use(helmet());
app.use(cors({ origin: process.env.FRONTEND_URL || 'http://localhost:3001' }));
app.use(morgan('dev'));
app.use(express.json());

// Routes mounted here in later chapters
// app.use('/api/auth', authRouter);
// app.use('/api/courses', coursesRouter);

// Error handler last
import { errorHandler } from './middleware/errorHandler.js';
app.use(errorHandler);

export default app;
```

**src/server.js** — start the server:

```js
// src/server.js — shape only
import app from './app.js';

const PORT = process.env.PORT || 3000;

app.listen(PORT, () => {
  console.log(`EduFlow server running on port ${PORT}`);
});
```

For now, server.js is minimal — no database connection yet. You will add config validation and the Mongoose connection in chapter 04.

✅ npm run dev should start without errors and print the port message.
❌ Do not put business logic in app.js or server.js. They are wiring, not feature code.
