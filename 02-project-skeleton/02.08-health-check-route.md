# The Health Check Route

A health check is the simplest useful route an API can have. It proves the server is running and reachable before you invest time in any feature. Add it directly in app.js for now:

```js
// In app.js, after middleware and before routes
app.get('/api/health', (req, res) => {
  res.json({ status: 'ok', timestamp: new Date().toISOString() });
});
```

Test it immediately:

```
curl http://localhost:3000/api/health
```

Expected output:

```json
{ "status": "ok", "timestamp": "2026-02-15T10:00:00.000Z" }
```

✅ You see the JSON response above.
❌ If you see "Cannot GET /api/health", the route was not mounted — check the path spelling and ensure app.js is exporting correctly.

This route stays for the life of the project. It is how your load balancer, monitoring system, and your own sanity check confirm the server is alive.
