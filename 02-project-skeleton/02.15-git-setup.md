# Git Setup

With two applications in one repository, a clean root .gitignore prevents accidents:

```
# Root .gitignore
node_modules
.env
.env.*
.env.local
.env*.local
dist
.next
.DS_Store
```

**Monorepo structure:** the root holds docker-compose.yml and both app folders. You can run git commands from the root and they cover both apps. Each app also has its own local .gitignore for app-specific ignores.

**Initial commit from the root:**

```
cd /path/to/eduflow
git init  # if not done yet
git add .
git status  # confirm no .env files appear
git commit -m "chore: project skeleton — server and client scaffolded"
git remote add origin https://github.com/your-username/eduflow.git
git push -u origin main
```

✅ git log shows one commit: the skeleton.
✅ git status shows nothing untracked after the commit.
❌ If you see .env in git status before committing, run: git restore --staged .env
