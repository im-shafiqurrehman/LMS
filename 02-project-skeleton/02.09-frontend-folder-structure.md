# Frontend Folder Structure

The client/ directory holds the Next.js application. Before you initialise it, understand the structure you are building toward:

```
client/
├── app/
│   ├── layout.tsx              ← root layout, wraps everything with Redux Provider
│   ├── page.tsx                ← homepage
│   ├── courses/
│   │   ├── page.tsx            ← course catalogue
│   │   └── [id]/page.tsx       ← course detail
│   ├── dashboard/
│   │   ├── page.tsx            ← student dashboard
│   │   └── instructor/page.tsx ← instructor dashboard
│   ├── login/page.tsx
│   └── register/page.tsx
├── components/
│   ├── Providers.tsx           ← Redux Provider wrapper (must be a client component)
│   └── Navbar.tsx
├── lib/
│   └── api.ts                  ← axios instance with auth interceptor
├── store/
│   ├── store.ts                ← configureStore
│   ├── authSlice.ts            ← user, accessToken, isAuthenticated
│   └── courseSlice.ts          ← catalogue list, loading, error
├── types/
│   └── index.ts                ← TypeScript interfaces matching API response shapes
└── .env.local
```

You will create this structure in 02.12. First, initialise the app.

Key decisions baked into this structure:

- **store/** owns all global state — auth and course catalogue data. No other state library.
- **lib/api.ts** is the single axios instance. Every API call goes through it so auth headers are added automatically.
- **Providers.tsx** is a client component that wraps the Redux Provider. The root layout uses it to make the store available everywhere.
