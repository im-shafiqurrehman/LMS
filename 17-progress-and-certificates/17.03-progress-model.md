# Progress Model Deep Dive

The ProgressRecord schema was defined in chapter 05. Here is how it is queried at scale.

For the student dashboard (all enrollments, all courses), you need progress data for multiple courses in one query. Instead of N queries (one per enrollment), use a single aggregation:

```js
const progress = await ProgressRecord.aggregate([
  { $match: { enrollmentId: { $in: enrollmentIds } } },
  { $group: {
    _id:   '$enrollmentId',
    count: { $sum: 1 },
  }},
]);
```

This returns the count of completed lessons per enrollment in one query — not N queries.

An index on { enrollmentId: 1 } (from chapter 05) makes this aggregation efficient regardless of the number of ProgressRecord documents.
