# The .env File

Create server/.env. This file never enters Git. Populate the variables you need for this chapter:

```
# server/.env
NODE_ENV=development
PORT=3000

DATABASE_URL=mongodb://admin:yourpassword@localhost:27017/eduflow?authSource=admin

JWT_SECRET=replace-me-with-a-minimum-32-char-string-here
JWT_REFRESH_SECRET=replace-me-with-a-different-32-char-string

JWT_ACCESS_EXPIRY=15m
JWT_REFRESH_EXPIRY=7d

FRONTEND_URL=http://localhost:3001
```

Leave Stripe, Cloudinary, Redis, and Resend blank — temporarily mark them optional in config/index.js with a TODO comment noting which chapter makes them required.

**For production**, you will never have a .env file on the server — secrets are injected as operating-system environment variables. The .env file is a development convenience only. The config module does not care how the variables are set, only that they are.

**Security checks:**

```
cat server/.env | grep -i password   # should only show local dev values
git status                           # .env must not appear
git log --oneline | head -5          # .env must never have been committed
```

✅ All three checks pass.
✅ The server starts successfully with the values in .env.
