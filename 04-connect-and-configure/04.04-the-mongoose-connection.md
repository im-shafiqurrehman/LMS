# The Mongoose Connection

Create server/src/lib/db.js. This file establishes the Mongoose connection and exports a connectDB() function that server.js calls at boot.

```js
// src/lib/db.js — shape only
import mongoose from 'mongoose';
import { config } from '../config/index.js';

let isConnected = false;

export async function connectDB() {
  if (isConnected) return;  // prevent duplicate connections in hot-reload

  const conn = await mongoose.connect(config.databaseUrl);
  isConnected = true;
  console.log(`MongoDB connected: ${conn.connection.host}`);
}
```

**Why the isConnected guard?** In development, nodemon restarts the server on file changes. Without the guard, each restart would open a new Mongoose connection while the old one is still alive. Mongoose's internal connection state handles this, but the explicit guard is clearer and prevents the warning logs.

**Why connectDB() instead of connecting in app.js?** Separating the connection from the Express app setup makes the app testable — you can import app.js in tests without triggering a database connection.

> Worth reading: search "Mongoose connection best practices Node.js" — the Mongoose documentation has a section on connection management, including connection events and retry logic for production.
