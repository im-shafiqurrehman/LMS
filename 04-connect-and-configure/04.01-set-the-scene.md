# Connect and Configure — Set the Scene

The project skeleton in chapter 02 started the server, but it skipped two critical things: validating config at startup and connecting to the database. This chapter fixes both.

The pattern goes like this: someone initialises a Node project, writes a server.js that reads process.env.DATABASE_URL directly in a route file, and pushes to GitHub with a real database password in the commit history. Three weeks later, config is scattered across ten files, secrets are in git log, and nobody can find where a variable is read. None of this is hard to prevent — you prevent it by making two decisions at the start:

1. **All config is read from the environment, in one place, at startup.** If any required variable is missing or malformed, the server refuses to start with a clear error. This is "fail fast" — better to crash at boot with a readable message than to crash at 3am during a request because JWT_SECRET is undefined.
2. **.env never enters Git.** Not once. Not "just temporarily."

By the end of this chapter:
- The server reads all config through src/config/index.js
- Missing or malformed env variables cause an immediate, readable error at startup
- Mongoose is connected and the server prints the connection host on startup
- GET /api/health still works
