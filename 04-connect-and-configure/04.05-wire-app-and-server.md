# Wire app.js and server.js

Now update server.js to connect the database before the server starts listening. The ordering is critical — requests that arrive before the database is connected will fail.

```js
// src/server.js — final shape for this chapter
import 'dotenv/config';
import { config }    from './config/index.js';
import { connectDB } from './lib/db.js';
import app           from './app.js';

async function start() {
  await connectDB();                     // database first
  app.listen(config.port, () => {
    console.log(`EduFlow running on port ${config.port} [${config.nodeEnv}]`);
  });
}

start().catch(err => {
  console.error('Failed to start:', err);
  process.exit(1);
});
```

**The critical ordering:** connectDB() resolves before app.listen() is called. If the database is unreachable, process.exit(1) prevents the server from accepting requests it cannot service.

**Update app.js** to import config (this triggers validation at module load time — if config fails, the app never starts):

```js
// src/app.js — add at the top
import { config } from './config/index.js';
```

Start the server:

```
npm run dev
```

Expected console output (in order):
1. MongoDB connected: localhost
2. EduFlow running on port 3000 [development]

✅ Both lines appear.
❌ If you see "MongoNetworkError", confirm docker compose ps shows mongodb as running and the DATABASE_URL matches the credentials in docker-compose.yml.
