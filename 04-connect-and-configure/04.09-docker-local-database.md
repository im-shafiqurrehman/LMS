# Verify the Docker Local Database

If you have not run docker compose up -d from chapter 02, do it now. This chapter's Mongoose connection needs MongoDB to be running.

```
cd /path/to/eduflow
docker compose up -d
docker compose ps
```

Both services (mongodb and redis) should show as running.

**Verify MongoDB is reachable:**

```
docker compose exec mongodb mongosh -u admin -p yourpassword \
  --eval "db.adminCommand('ping')"
```

Expected: { ok: 1 }

**Verify Mongoose connects from Node:**

Start the server with npm run dev. You should see:

```
MongoDB connected: localhost
EduFlow running on port 3000 [development]
```

If you see a MongoNetworkError instead:
1. Check docker compose ps — is mongodb running?
2. Check DATABASE_URL in .env — do the credentials match docker-compose.yml?
3. Check the port — 27017 must not be in use by another process

> Worth reading: search "Docker Compose networking explained" — understanding why localhost:27017 works to reach a container (vs docker's internal DNS) helps when you move to production where the database runs on a different host.
