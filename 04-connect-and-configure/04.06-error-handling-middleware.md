# Error Handling Middleware

Create server/src/middleware/errorHandler.js. Every unhandled error in the application should end up here and return a consistent JSON shape:

```js
// src/middleware/errorHandler.js — shape only
export function errorHandler(err, req, res, next) {
  const status  = err.statusCode || err.status || 500;
  const message = err.message    || 'Internal server error';

  // Never expose stack traces in production
  const body = { error: message };
  if (process.env.NODE_ENV !== 'production') {
    body.stack = err.stack;
  }

  res.status(status).json(body);
}
```

Wire it in app.js after all routes — Express identifies a four-argument function as an error handler:

```js
// src/app.js — error handler goes last
app.use(errorHandler);
```

**Optional: create a small AppError class** to attach statusCode to thrown errors:

```js
// src/lib/AppError.js
export class AppError extends Error {
  constructor(message, statusCode = 500) {
    super(message);
    this.statusCode = statusCode;
    Error.captureStackTrace(this, this.constructor);
  }
}
```

Usage: throw new AppError('Course not found', 404) in service functions. The error handler catches it and returns the correct status code.

✅ A deliberate throw new AppError('Test error', 418) in a route returns { "error": "Test error" } with HTTP 418.
✅ A deliberate throw new Error('Unexpected') returns HTTP 500 with the message.
