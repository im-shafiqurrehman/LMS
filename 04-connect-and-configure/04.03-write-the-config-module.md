# Write the Config Module

Open server/src/config/index.js and implement the full config module. The complete shape it must produce:

```js
// Result after validation — this is what the rest of the app imports
{
  port:              3000,
  nodeEnv:           'development',
  databaseUrl:       'mongodb://admin:...',
  jwtSecret:         '...',
  jwtRefreshSecret:  '...',
  jwtAccessExpiry:   '15m',
  jwtRefreshExpiry:  '7d',
  frontendUrl:       'http://localhost:3001',
  // These are optional until their chapters:
  // cloudinaryCloudName, cloudinaryApiKey, cloudinaryApiSecret
  // redisUrl
  // resendApiKey
  // stripeSecretKey, stripeWebhookSecret
}
```

For optional variables (Cloudinary, Redis, Resend, Stripe), mark them with .optional() for now and add a TODO comment noting which chapter makes them required:

```js
// TODO: required before chapter 18 (Redis)
REDIS_URL: z.string().optional(),
// TODO: required before chapter 13 (Stripe)
STRIPE_SECRET_KEY: z.string().optional(),
```

When a chapter arrives that needs one of these, remove .optional() — the server will then refuse to start without it, which is exactly what you want.

**Important:** import dotenv/config (or dotenv.config()) at the very top of server.js, before importing config/index.js. The config module reads process.env — dotenv must populate it first.

```js
// src/server.js — ordering matters
import 'dotenv/config';              // populates process.env from .env
import { config } from './config/index.js';  // validates
import app from './app.js';
```
