# When to Embed, When to Reference

You reviewed the theory in chapter 03. Here is how it applies to EduFlow's specific models.

**EduFlow uses references throughout.** Every relationship is expressed as an ObjectId pointing to another document's _id.

The test for each relationship:

| Relationship | Embed or Reference? | Why |
|---|---|---|
| Course → Instructor | Reference (instructorId) | User is queried independently, referenced by many documents |
| Course → Lessons | Reference (each Lesson has courseId) | Lessons are queried independently for the lesson viewer |
| Enrollment → Course | Reference (courseId) | Course is queried independently for the catalogue |
| Enrollment → Student | Reference (studentId) | User is referenced by many documents |
| ProgressRecord → Lesson | Reference (lessonId) | Lessons are queried independently |
| Certificate → Enrollment | Reference (enrollmentId) | Enrollment is queried independently |
| Question → Course | Reference (courseId) | Course is queried independently |

**Could lessons be embedded inside courses?** Technically yes — if you always fetch all lessons when you fetch a course, and lessons are never queried on their own. But the lesson viewer queries a single lesson directly (GET /api/lessons/:id), so lessons must be independently addressable. Reference.

**Could category tags be embedded on Course?** Yes — a course has a categories array of strings (e.g. ["programming", "web-dev"]). These are not separate documents; they are facts about the course. They never need to be queried independently. Embed.
