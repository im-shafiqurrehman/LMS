# Verify Your Models

Before moving to authentication, verify that every model is correctly defined.

**1. Create the barrel file** server/src/models/index.js:

```js
// src/models/index.js
export { User }           from './user.model.js';
export { Category }       from './category.model.js';
export { Course }         from './course.model.js';
export { Lesson }         from './lesson.model.js';
export { Enrollment }     from './enrollment.model.js';
export { ProgressRecord } from './progressRecord.model.js';
export { Certificate }    from './certificate.model.js';
export { Question }       from './question.model.js';
export { Answer }         from './answer.model.js';
export { RefreshToken }   from './refreshToken.model.js';
```

**2. Run the seed script** and verify in mongosh:

```
npm run seed
# Expected output: Seed complete
```

**3. Check collections exist:**

```
mongosh "mongodb://admin:yourpassword@localhost:27017/eduflow?authSource=admin"
> use eduflow
> show collections
# Should list: users, categories, courses, lessons
```

**4. Verify the password hook worked:**

```js
> db.users.findOne({ email: 'admin@eduflow.dev' }, { passwordHash: 1 })
// passwordHash should start with $2b$ (bcrypt hash), not be "Admin1234!"
```

**5. Verify indexes were created:**

```js
> db.courses.getIndexes()
// Should show: status+createdAt, categoryId+status, instructorId
```

✅ All checks pass.
❌ If passwordHash is plain text, the pre('save') hook is not wired correctly — check that you are calling user.save() not User.insertOne().
