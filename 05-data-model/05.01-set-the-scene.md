# Data Model — Set the Scene

Every feature in EduFlow depends on the shape of the data underneath it. Get the model right and features slot in cleanly. Get it wrong and you spend the rest of the course fighting the database.

This chapter defines all ten Mongoose model schemas for EduFlow. You will write the actual schema files — not placeholders — and seed initial data to verify they work. By the end, the database has real documents you can query, and every model is imported through a central barrel file.

---

## What this chapter covers

- User model (with bcrypt hook and select:false security fields)
- Course, Category, and Lesson models
- Enrollment model (with the price snapshot decision)
- ProgressRecord and Certificate models
- Question and Answer models
- RefreshToken model (with TTL index for automatic expiry)
- Indexes on every model that will be queried at scale
- A seed script that populates test data
- Verification using mongosh

No authentication logic, no routes — just the data foundation.
