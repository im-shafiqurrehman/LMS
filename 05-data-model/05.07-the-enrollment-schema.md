# The Enrollment Schema

The Enrollment document records the fact that a student paid to access a course. The most important field is amountPaid — the price snapshot.

```js
// src/models/enrollment.model.js — shape only
import mongoose from 'mongoose';

const enrollmentSchema = new mongoose.Schema(
  {
    studentId: {
      type:     mongoose.Schema.Types.ObjectId,
      ref:      'User',
      required: true,
    },
    courseId: {
      type:     mongoose.Schema.Types.ObjectId,
      ref:      'Course',
      required: true,
    },
    amountPaid: {
      type:     Number,
      required: true,
      min:      0,
    },
    stripePaymentIntentId: {
      type:   String,
      unique: true,
    },
    enrolledAt: {
      type:    Date,
      default: Date.now,
    },
  },
  { timestamps: true }
);

// A student cannot enroll in the same course twice
enrollmentSchema.index({ studentId: 1, courseId: 1 }, { unique: true });
enrollmentSchema.index({ studentId: 1 });
enrollmentSchema.index({ courseId: 1 });

export const Enrollment = mongoose.model('Enrollment', enrollmentSchema);
```

**The compound unique index** on { studentId, courseId } prevents duplicate enrollments at the database level — even if two concurrent requests try to create the same enrollment, only one will succeed.

**stripePaymentIntentId** is how you detect duplicate Stripe webhook events (idempotency) — covered in chapter 13.
