# Indexes — What, Why, When

You have added indexes inline with each schema. This sub-chapter explains the reasoning.

**What an index is:** a B-tree data structure that MongoDB maintains alongside a collection. Without an index, MongoDB scans every document (a collection scan) to satisfy a query. With an index on the queried field, it jumps directly to the matching documents.

**When to add an index:** before the collection is large. Adding an index to a large collection is a slow, blocking operation. Add the indexes now, while the collections are empty.

**EduFlow's complete index list:**

| Collection | Index | Reason |
|---|---|---|
| courses | { status, createdAt } | Catalogue query: filter published, sort by newest |
| courses | { categoryId, status } | Catalogue filter by category |
| courses | { instructorId } | "My courses" for instructors |
| lessons | { courseId, position } | Lesson list in position order |
| enrollments | { studentId, courseId } unique | Prevent duplicate enrollments |
| enrollments | { studentId } | "My enrollments" for students |
| enrollments | { courseId } | Course enrollment count for admin |
| progressRecords | { enrollmentId, lessonId } unique | Progress lookup |
| questions | { courseId, createdAt } | Q&A list by course |
| answers | { questionId } | Answers for a question |
| refreshTokens | { userId } | Revoke on logout |
| refreshTokens | { expiresAt } TTL | Auto-delete expired tokens |

✅ All indexes are declared before the mongoose.model() call in each schema file.
❌ Do not add indexes after the fact on large collections — it is slow and can cause performance issues.
