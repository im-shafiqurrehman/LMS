# Basic Monitoring

Production monitoring is not optional — you need to know when something breaks before your users do.

**PM2 built-in monitoring:**
```bash
pm2 monit    # real-time CPU, memory, restart count for each process
```

**Nginx access logs:**
```bash
tail -f /var/log/nginx/access.log    # every request
tail -f /var/log/nginx/error.log     # errors
```

**MongoDB Atlas monitoring:** Atlas provides query performance, connection counts, and storage usage in the web UI.

**Bar-raiser options:**
- UptimeRobot (free) — monitors your URL every 5 minutes and alerts by email on downtime
- Sentry — JavaScript error tracking with stack traces; integrates with both Express and Next.js
- Datadog — full observability stack (metrics, logs, traces)
