# Keyword Search Input

Add a debounced search input to CoursesPage. Debouncing prevents a new API call on every keystroke.

```tsx
// Install: npm install use-debounce
import { useDebounce } from 'use-debounce';

// Inside CoursesPage:
const [rawQ, setRawQ] = useState('');
const [q]             = useDebounce(rawQ, 400);  // wait 400ms after last keystroke

// useEffect depends on q (debounced), not rawQ
useEffect(() => { fetchCourses({ reset: true }); }, [categoryId, q]);
```

Or implement debounce manually without a library:

```ts
// lib/debounce.ts
export function debounce<T extends (...args: any[]) => any>(fn: T, delay: number) {
  let timeout: ReturnType<typeof setTimeout>;
  return (...args: Parameters<T>) => {
    clearTimeout(timeout);
    timeout = setTimeout(() => fn(...args), delay);
  };
}
```

Add the input to the page layout above the course grid:

```tsx
<input
  value={rawQ}
  onChange={e => setRawQ(e.target.value)}
  placeholder="Search courses..."
  style={{ padding: '0.5rem', width: '100%', marginBottom: '1rem' }}
/>
```

✅ Typing "react" in the search input (and pausing) triggers a search request.
✅ The URL is not updated by the search — that is a bar-raiser improvement (add router.push with query params).
