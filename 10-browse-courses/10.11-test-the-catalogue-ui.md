# Test the Catalogue UI

Walk through the full catalogue in the browser.

**Test 1 — Course grid:**
1. Open http://localhost:3001/courses
2. The course grid shows cards from the seeded data
3. Each card shows title, description preview, price, and lesson count

**Test 2 — Category filter:**
1. Click "Web Development" in the category filter
2. The grid refreshes showing only web development courses
3. Click "All" — all courses appear again

**Test 3 — Keyword search:**
1. Type "python" in the search input (pause 400ms)
2. The grid shows only matching courses

**Test 4 — Load More:**
1. Reduce the limit to 3 in the API call temporarily
2. Verify the Load More button appears and appends courses

**Test 5 — Course detail:**
1. Click "View Course" on any card
2. The detail page shows the course title, description, lessons list
3. No video URL appears in the lesson list

**Test 6 — Browser DevTools:**
Open Network tab and verify:
- Only 1 request is made per page load (not N+1)
- The populate() is working server-side (instructor email appears in the response)

✅ All six tests pass.
