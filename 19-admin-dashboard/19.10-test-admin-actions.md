# Test Admin Actions

**Test 1 — Approve a course:**
1. Submit a course for review as an instructor
2. Log in as admin
3. GET /api/admin/courses/pending — the course should appear
4. PATCH /api/admin/courses/:id/approve — approve it
5. GET /api/courses — the course should appear in the public catalogue

**Test 2 — Reject a course:**
1. Submit another course for review
2. PATCH /api/admin/courses/:id/reject with { rejectionReason: 'Missing lessons' }
3. As instructor: GET /api/instructor/courses/mine — status should be 'rejected' with the reason

**Test 3 — Metrics:**
GET /api/admin/metrics returns correct counts.

**Test 4 — Access control:**
Instructor or student token on /api/admin/* returns 403.
