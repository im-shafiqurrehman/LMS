# Recap and What Is Next

This chapter added two access control layers to the backend:

- **authorize(role) middleware** — checks req.user.role before the handler runs; returns 403 if insufficient
- **Ownership checks pattern** — each service function that modifies a resource checks instructorId === userId before performing the operation

Together they prevent:
- Students accessing instructor-only routes (RBAC)
- Instructor A modifying Instructor B's courses (IDOR)

What is next: Chapter 09 — Course Catalogue API. The most-visited feature of any LMS is the course catalogue. This chapter builds the read-heavy API: list published courses with filtering, sorting, keyword search, and cursor pagination.
