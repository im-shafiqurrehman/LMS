# Test Role Rejection

Before moving on, verify every failure case in the authorization chain.

```bash
BASE=http://localhost:3000

# Step 1: Login as student — get token
STUDENT_TOKEN=$(curl -s -X POST $BASE/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"student@test.com","password":"Test1234!"}' \
  | python3 -c "import sys,json;print(json.load(sys.stdin)['accessToken'])")

# Step 2: No token → 401
curl -s $BASE/api/auth/me
# Expected: 401

# Step 3: Student token on admin route → 403
curl -s $BASE/api/admin/test -H "Authorization: Bearer $STUDENT_TOKEN"
# Expected: 403 {"error":"Insufficient permissions"}

# Step 4: Valid but expired token → 401
# (Use a token that was issued more than 15 minutes ago — or temporarily reduce JWT_ACCESS_EXPIRY to "5s")

# Step 5: Completely fake token → 401
curl -s $BASE/api/auth/me -H "Authorization: Bearer fake.token.here"
# Expected: 401

# Step 6: Login as admin → 200 on admin route
ADMIN_TOKEN=$(curl -s -X POST $BASE/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@eduflow.dev","password":"Admin1234!"}' \
  | python3 -c "import sys,json;print(json.load(sys.stdin)['accessToken'])")
curl -s $BASE/api/admin/test -H "Authorization: Bearer $ADMIN_TOKEN"
# Expected: 200
```

✅ All six tests return the expected responses.
