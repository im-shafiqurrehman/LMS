# Authorization — Set the Scene

Authentication answers "Who are you?" Authorization answers "What are you allowed to do?"

EduFlow has three user roles: student, instructor, admin. The same GET /api/courses endpoint is public. POST /api/courses/create is instructor-only. POST /api/admin/courses/:id/approve is admin-only. Authentication identifies the user; authorization enforces what they can access.

This chapter adds two layers of access control:

1. **Role-based access control (RBAC):** an authorize(role) middleware that checks req.user.role against the allowed roles for a route
2. **Ownership checks:** service-layer logic that verifies a user can only modify their own resources (an instructor cannot edit another instructor's course)

Both layers are essential. Without RBAC, a student can reach instructor routes. Without ownership checks, Instructor A can edit Instructor B's course. This second type of vulnerability is called an Insecure Direct Object Reference (IDOR).
