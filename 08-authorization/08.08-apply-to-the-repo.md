# Apply Authorization to the Repo

With the authorize middleware ready, apply it to the auth routes that already exist. This is also a preview of how every future module will use it.

**In server/src/modules/auth/auth.routes.js**, confirm that:
- GET /api/auth/me uses authenticate (it already does)
- No auth route needs authorize — authentication is a public-facing feature

**Create a test admin user** using the seed script (if not already done in chapter 05). The admin user is the first account you will use to approve courses in chapter 19.

**Verify the middleware chain works end-to-end:**

1. Register a test student account
2. Log in and get the access token
3. Try to access an admin-only route (you can add a test route temporarily):

```js
// app.js — temporary test route
app.get('/api/admin/test', authenticate, authorize('admin'), (req, res) => {
  res.json({ message: 'Admin access granted', user: req.user });
});
```

```
curl http://localhost:3000/api/admin/test \
  -H "Authorization: Bearer YOUR_STUDENT_TOKEN"
```

Expected: 403 { "error": "Insufficient permissions" }

Login as admin and repeat — expected: 200.

Remove the test route. Commit the authorize middleware:

```
git add server/src/middleware/authorize.js
git commit -m "feat(auth): add authorize RBAC middleware"
```
