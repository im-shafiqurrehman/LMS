# Ownership Checks in Services

The authorize middleware handles role-based access. Ownership checks are the responsibility of individual service functions.

**The pattern:**

```js
// src/modules/courses/courses.service.js — shape only
async function updateCourse(courseId, userId, role, updates) {
  const course = await Course.findById(courseId);
  if (!course) throw new AppError('Course not found', 404);

  // Admin bypass: admin can edit any course
  if (role !== 'admin' && course.instructorId.toString() !== userId) {
    throw new AppError('Forbidden', 403);
  }

  Object.assign(course, updates);
  return course.save();
}
```

The service receives userId and role from req.user (set by authenticate middleware). It fetches the resource, then checks ownership before modifying it. Admin bypass is explicit — if you forget to add it, admins cannot moderate courses.

**Consistent pattern across all write operations:**

| Service function | Ownership check |
|---|---|
| updateCourse | course.instructorId === userId || role === 'admin' |
| deleteCourse | course.instructorId === userId (no admin bypass — only owner can delete) |
| addLesson | course.instructorId === userId |
| deleteLesson | course.instructorId === userId |

Decide the admin bypass policy for each operation deliberately — do not apply it universally.
