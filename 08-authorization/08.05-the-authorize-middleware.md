# The Authorize Middleware

Create server/src/middleware/authorize.js. The authorize function returns a middleware that checks req.user.role:

```js
// src/middleware/authorize.js
export function authorize(...roles) {
  return (req, res, next) => {
    if (!req.user) {
      return res.status(401).json({ error: 'Not authenticated' });
    }
    if (!roles.includes(req.user.role)) {
      return res.status(403).json({ error: 'Insufficient permissions' });
    }
    next();
  };
}
```

Usage:

```js
// Admin only
router.get('/admin/users', authenticate, authorize('admin'), listUsers);

// Instructor or admin
router.post('/courses', authenticate, authorize('instructor', 'admin'), createCourse);

// Any authenticated user
router.get('/me', authenticate, getMe);
```

The ...roles spread parameter means you can pass one or multiple allowed roles:
- authorize('admin') — admin only
- authorize('instructor', 'admin') — instructor or admin
- authorize('student', 'instructor', 'admin') — any authenticated user (same as just using authenticate)

✅ As a student, try to POST /api/courses — expect 403.
✅ As an instructor, try to GET /api/admin/users — expect 403.
✅ As an admin, both routes are accessible.
