# The EduFlow Role Matrix

A role matrix documents what each role can do. Write this before implementing routes — it prevents the "I'll figure out permissions as I go" trap.

| Resource | Guest | Student | Instructor | Admin |
|----------|-------|---------|-----------|-------|
| Register / Login | ✅ | ✅ | ✅ | ✅ |
| Browse published courses | ✅ | ✅ | ✅ | ✅ |
| View lesson (enrolled) | ❌ | ✅ | ✅ | ✅ |
| Enroll in a course | ❌ | ✅ | ✅ | ❌ |
| Create a course | ❌ | ❌ | ✅ | ✅ |
| Edit own course | ❌ | ❌ | ✅ own | ✅ any |
| Delete own course | ❌ | ❌ | ✅ own | ✅ any |
| Submit course for review | ❌ | ❌ | ✅ own | ❌ |
| Approve / reject course | ❌ | ❌ | ❌ | ✅ |
| View platform metrics | ❌ | ❌ | ❌ | ✅ |
| Manage users | ❌ | ❌ | ❌ | ✅ |
| Post a question | ❌ | ✅ | ✅ | ✅ |
| Answer a question | ❌ | ❌ | ✅ own course | ✅ |

✅ Print this and keep it next to your IDE while implementing the remaining chapters.
❌ Do not decide permissions at implementation time — you will forget edge cases.
