# Test the Auth UI

Walk through the complete authentication flow in the browser before marking this chapter done.

**Test 1 — Registration and verification:**
1. Open http://localhost:3001/register
2. Enter a new email and a password that meets the requirements (8+ chars, one uppercase, one number)
3. Submit. You should navigate to /verify
4. Check the server console for the OTP
5. Enter the OTP on /verify and submit
6. You should navigate to /login

**Test 2 — Login and protected route:**
1. Log in with the registered credentials
2. You should navigate to /dashboard
3. The Navbar shows your email and a Dashboard link
4. Open a new tab and navigate to /dashboard — you should still be logged in

**Test 3 — Page refresh:**
1. While logged in, press F5 to refresh
2. The auth state should rehydrate (you may see a brief flicker while the refresh token call completes)
3. The Navbar should show the logged-in state after rehydration

**Test 4 — Logout:**
1. Click Logout in the Navbar
2. You should navigate to /login
3. Trying to navigate to /dashboard redirects back to /login

✅ All four tests pass.
❌ If the refresh does not rehydrate auth state, check Providers.tsx — the AuthRehydrator effect may not be running.
