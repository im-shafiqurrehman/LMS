# What You Will Build

The frontend authentication flow has five screens and a cross-cutting concern:

**Screens:**
1. Register page — email + password form; calls POST /api/auth/register
2. Verify email page — OTP input; calls POST /api/auth/verify-email
3. Login page — email + password form; calls POST /api/auth/login, dispatches setCredentials
4. Dashboard home (protected) — shows the user's email and role
5. Logout — dispatches logout action, redirects to /login

**Cross-cutting concern — auth guard:** a higher-order component (or Next.js middleware) that redirects unauthenticated users trying to access protected pages.

Redux slices involved:
- authSlice: setCredentials, logout, and the auth state shape
- No new slice needed — authSlice from chapter 02 covers everything
