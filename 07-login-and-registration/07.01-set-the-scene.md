# Login and Registration — Set the Scene

The backend authentication API is complete. This chapter builds the frontend: the registration form, email verification screen, login form, protected route guards, and logout. By the end, a user can register, verify their email, log in, and see their dashboard — all in the browser.

State management uses Redux Toolkit (the authSlice you scaffolded in chapter 02). There is no local state for user identity — it all lives in the Redux store.

What this chapter produces:
- /register — registration form
- /verify — OTP entry form  
- /login — login form
- Auth state persisted across page navigations (via sessionStorage)
- Protected routes that redirect unauthenticated users to /login
- Logout that clears the store and redirects to /login
