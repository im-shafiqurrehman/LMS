# Persist Auth State Across Navigations

Redux state is in-memory. A page refresh loses it. For the access token, this is fine — it is short-lived and the user can refresh it. But it means the user is "logged out" on every page refresh until the app re-fetches the token.

For EduFlow (a learning project), the simplest solution is:
- Store the refresh token in sessionStorage at login
- On app load (in Providers.tsx), if a refresh token is found, call POST /api/auth/refresh to get a new access token and rehydrate the Redux store

```tsx
// components/Providers.tsx — updated
'use client';
import { useEffect }   from 'react';
import { Provider }    from 'react-redux';
import { store }       from '@/store/store';
import { setCredentials } from '@/store/authSlice';
import api from '@/lib/api';

function AuthRehydrator() {
  useEffect(() => {
    const refreshToken = sessionStorage.getItem('refreshToken');
    if (!refreshToken) return;

    api.post('/api/auth/refresh', { refreshToken })
      .then(({ data }) => {
        return api.get('/api/auth/me', {
          headers: { Authorization: `Bearer ${data.accessToken}` }
        }).then(({ data: user }) => {
          store.dispatch(setCredentials({ user, accessToken: data.accessToken }));
        });
      })
      .catch(() => {
        sessionStorage.removeItem('refreshToken'); // expired or revoked
      });
  }, []);
  return null;
}

export function Providers({ children }: { children: React.ReactNode }) {
  return (
    <Provider store={store}>
      <AuthRehydrator />
      {children}
    </Provider>
  );
}
```

✅ After login, refresh the page — the user stays logged in.
✅ After logout (next sub-chapter), refresh the page — the user is correctly logged out.
