# Certificate PDF Generation (Stub)

A real certificate is a PDF with the student's name, course title, and date. For now, implement as a stub that returns the Certificate document without a PDF.

The bar-raiser version uses a PDF generation library:

**Option 1:** puppeteer — renders an HTML template to PDF (heavy, requires Chrome)
**Option 2:** pdfkit — generates PDF programmatically from JavaScript (lightweight)
**Option 3:** html-pdf-node — similar to puppeteer but simpler API

For any option, the flow is:
1. Generate the PDF as a Buffer
2. Upload the Buffer to Cloudinary or S3 as a private asset
3. Store the resulting URL in certificate.pdfUrl
4. Return the signed URL to the client

Document the stub clearly:
```js
// TODO: generate PDF and upload — pdfUrl is null until then
```

✅ POST /api/learn/certificates/:courseId returns a Certificate document (pdfUrl may be null).
✅ Calling twice returns the same certificate.
