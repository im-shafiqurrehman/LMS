# Signed URLs for Video

Video URLs in the Lesson documents can be either:

1. **Public CDN URLs** — works for getting started; anyone with the URL can access the video
2. **Signed URLs** — short-lived URLs that expire after a few minutes; only the enrolled student who requested the URL can use it

For a production LMS, signed URLs are required — you cannot let unenrolled students share video links.

EduFlow uses public URLs for now (the videoUrl is a direct string). The signed URL pattern is:

1. Video is stored in S3 (or Cloudinary private storage)
2. The API generates a signed URL on each request with a 1-hour TTL
3. The student's browser accesses the signed URL directly — the server is not a streaming proxy

The stub: the lesson API returns videoUrl directly. A bar-raiser implementation replaces this with S3 presigned URL generation using the AWS SDK.

> Worth reading: search "AWS S3 presigned URL Node.js" or "Cloudinary private video URL signed token".
