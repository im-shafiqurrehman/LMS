# Document Your Project

Three documents every portfolio project needs:

**1. README.md** — the public face of the repo:
- What EduFlow is (2-3 sentences)
- Features list (bullet points)
- Tech stack table
- Setup instructions (clone, install, env, seed, dev)
- Screenshot or demo link

**2. Architecture Decision Record (ADR)**
One markdown file documenting your major technical decisions and why:
- Why MongoDB over PostgreSQL
- Why Redux Toolkit over React Query
- Why cursor pagination over offset
- Why two-token auth pattern

**3. Entity Relationship Diagram (ERD)**
A visual diagram of the Mongoose models and their relationships. Tools: draw.io, Mermaid in a .md file, or dbdiagram.io.
