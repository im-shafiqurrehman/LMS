# Enrollment Model (Revisited)

From chapter 05, the Enrollment schema:

```js
{
  studentId:            ObjectId ref User,
  courseId:             ObjectId ref Course,
  amountPaid:           Number,           // price snapshot
  stripePaymentIntentId: String unique,    // idempotency key
  enrolledAt:           Date,
}
// Compound unique index: { studentId, courseId }
```

The compound unique index prevents a student from enrolling in the same course twice at the database level — even if two concurrent webhook events arrive simultaneously.

stripePaymentIntentId unique index provides a second safety net: even if the studentId/courseId index somehow fails, the same payment intent cannot create two enrollments.
