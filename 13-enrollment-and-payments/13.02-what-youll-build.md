# What You Will Build

| Endpoint | Who | What |
|----------|-----|------|
| POST /api/enrollments/intent | student | Create a Stripe PaymentIntent |
| POST /api/enrollments/webhook | public (Stripe) | Handle payment confirmation |
| GET /api/enrollments/mine | student | List the student's enrollments |
| GET /api/enrollments/check/:courseId | student | Check if enrolled in a specific course |

The frontend (chapter 14) handles the Stripe Elements payment form. The backend receives the webhook and creates the enrollment.

**Critical decision:** enrollment is triggered by the webhook, not by the frontend. This means the enrollment survives even if the student closes the browser before the payment confirmation page loads.
