# Confirm Enrollment — Idempotency

The confirmEnrollment service function is called when a payment_intent.succeeded webhook arrives. It must be idempotent:

```js
async function confirmEnrollment(paymentIntent) {
  const { metadata, amount_received, id: stripePaymentIntentId } = paymentIntent;
  const { studentId, courseId } = metadata;

  // Check if enrollment already exists for this payment intent
  const existing = await Enrollment.findOne({ stripePaymentIntentId });
  if (existing) return; // Already processed — idempotent

  // Fetch course for price snapshot verification
  const course = await Course.findById(courseId);
  if (!course) throw new Error(`Course not found: ${courseId}`);

  await Enrollment.create({
    studentId,
    courseId,
    amountPaid:            amount_received / 100, // Stripe amounts are in cents
    stripePaymentIntentId,
    enrolledAt:            new Date(),
  });
}
```

The first line after the destructuring is the idempotency check: if an Enrollment for this stripePaymentIntentId already exists, return immediately. Stripe's webhook may deliver the same event multiple times — this is safe.
