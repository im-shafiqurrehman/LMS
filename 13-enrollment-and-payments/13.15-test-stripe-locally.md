# Test Stripe Locally with the CLI

Stripe provides a CLI for forwarding live webhooks to your local server:

```bash
# Install Stripe CLI: https://stripe.com/docs/stripe-cli
stripe listen --forward-to http://localhost:3000/api/enrollments/webhook
```

This gives you a webhook signing secret (whsec_...) — add it to server/.env as STRIPE_WEBHOOK_SECRET.

With the CLI running:
1. Make a payment in the frontend (chapter 14)
2. The CLI forwards the payment_intent.succeeded event to your server
3. Verify an Enrollment document was created in MongoDB

Alternative: trigger a test event directly from the Stripe CLI:
```bash
stripe trigger payment_intent.succeeded
```

This sends a synthetic webhook event — useful for testing the webhook handler without completing a full payment flow.
