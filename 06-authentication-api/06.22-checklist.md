# Chapter 06 — Definition of Done

- [ ] POST /api/auth/register creates a user; OTP is logged to console (email stub)
- [ ] POST /api/auth/verify-email marks the account verified when given the correct OTP
- [ ] POST /api/auth/verify-email returns 422 for wrong OTP, 409 for already-verified account
- [ ] POST /api/auth/login returns 200 with accessToken + refreshToken for a verified account
- [ ] POST /api/auth/login returns 401 for wrong password (same message as wrong email)
- [ ] POST /api/auth/login returns 403 for unverified account
- [ ] POST /api/auth/refresh returns a new accessToken for a valid refresh token
- [ ] POST /api/auth/refresh returns 401 for an invalid or deleted refresh token
- [ ] POST /api/auth/logout deletes the refresh token; a subsequent refresh returns 401
- [ ] GET /api/auth/me returns the user object for a valid access token
- [ ] GET /api/auth/me returns 401 for a missing, invalid, or expired access token
- [ ] RefreshToken documents in MongoDB have a TTL index on expiresAt
- [ ] passwordHash is never returned in any /api/auth response
- [ ] git commit with message: "feat(auth): registration, verification, login, refresh, logout"

All boxes ticked? Then continue to chapter 07 — Login and Registration (Frontend).
