# Verify Email Handler

POST /api/auth/verify-email accepts { email, otp } and activates the account.

The service must:

1. Find the user, explicitly selecting the hidden fields:
   User.findOne({ email }).select('+verificationOtp +otpExpiresAt')
   (select('+field') overrides the schema's select: false for this query)

2. If the user does not exist → 404.

3. If emailVerifiedAt is already set → 409 Conflict (account already verified).

4. If the OTP does not match or otpExpiresAt is in the past → 422 Unprocessable Entity.

5. Set emailVerifiedAt = new Date(), clear verificationOtp and otpExpiresAt, and call user.save().

6. Return { message: 'Email verified successfully.' }

```js
// auth.routes.js — add verify-email route
authRouter.post(
  '/verify-email',
  validate(verifyEmailSchema),
  async (req, res, next) => {
    try {
      await authService.verifyEmail(req.body);
      res.json({ message: 'Email verified successfully.' });
    } catch (err) {
      next(err);
    }
  }
);
```

✅ Verify the OTP from the register endpoint (check server console where it is logged).
✅ A second verify attempt returns 409.
✅ An expired OTP returns 422.
