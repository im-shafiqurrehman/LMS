# Refresh Token Handler

POST /api/auth/refresh accepts { refreshToken } and returns a new access token.

The service must:

1. Hash the incoming token: hashRefreshToken(req.body.refreshToken)

2. Look up the stored token: RefreshToken.findOne({ tokenHash })

3. If not found → 401.

4. If storedToken.expiresAt < new Date() → 401 (the TTL index may have already deleted it, but check in case of a race).

5. Look up the user: User.findById(storedToken.userId)

6. If user not found → 401.

7. Issue a new access token: issueAccessToken(user._id, user.role)

8. Return { accessToken }

**What NOT to do:** do not issue a new refresh token on every refresh call (token rotation). Rotation means the client must store the new refresh token on every refresh. For EduFlow's scope, the refresh token stays valid until logout or expiry. If you want rotation, add it as a bar-raiser feature.

✅ A valid refresh token returns a new access token.
✅ An invalid or expired refresh token returns 401.
✅ The original refresh token still works after calling /refresh (no rotation).
