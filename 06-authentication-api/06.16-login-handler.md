# Login Handler

POST /api/auth/login accepts { email, password } and returns both tokens.

The service must:

1. Find user, explicitly including passwordHash: User.findOne({ email }).select('+passwordHash')

2. If no user → 401 with "Invalid email or password." — not "email not found." Leaking which emails are registered is an **enumeration vulnerability**. The same generic message covers both cases.

3. Compare: await bcrypt.compare(password, user.passwordHash). If false → 401 same message.

4. If user.emailVerifiedAt is null → 403 "Please verify your email first."

5. Issue tokens:
   const rawRefreshToken  = issueRefreshToken();
   const accessToken      = issueAccessToken(user._id, user.role);
   const tokenHash        = hashRefreshToken(rawRefreshToken);

6. Store the hashed refresh token:
   await RefreshToken.create({
     tokenHash,
     userId:    user._id,
     expiresAt: new Date(Date.now() + 7 * 24 * 60 * 60 * 1000),
   });

7. Return { accessToken, refreshToken: rawRefreshToken, user: { id, email, role } }

✅ Login with correct credentials → 200 with both tokens.
✅ Login with wrong password → 401 "Invalid email or password."
✅ Login with unverified account → 403.
