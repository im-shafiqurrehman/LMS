# Test the Auth API with curl

Run through the full authentication flow from the command line before building the frontend.

```bash
BASE=http://localhost:3000

# 1. Register
curl -s -X POST $BASE/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"email":"alice@example.com","password":"Alice1234!"}'
# Expected: { "message": "Verification email sent..." }

# Check server console for the OTP (e.g., 482931)

# 2. Verify email
curl -s -X POST $BASE/api/auth/verify-email \
  -H "Content-Type: application/json" \
  -d '{"email":"alice@example.com","otp":"482931"}'
# Expected: { "message": "Email verified successfully." }

# 3. Login
TOKENS=$(curl -s -X POST $BASE/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"alice@example.com","password":"Alice1234!"}')
echo $TOKENS

ACCESS=$(echo $TOKENS | python3 -c "import sys,json;print(json.load(sys.stdin)['accessToken'])")
REFRESH=$(echo $TOKENS | python3 -c "import sys,json;print(json.load(sys.stdin)['refreshToken'])")

# 4. Protected route
curl -s $BASE/api/auth/me -H "Authorization: Bearer $ACCESS"
# Expected: user object

# 5. Refresh
curl -s -X POST $BASE/api/auth/refresh \
  -H "Content-Type: application/json" \
  -d "{\"refreshToken\":\"$REFRESH\"}"
# Expected: { "accessToken": "..." }

# 6. Logout
curl -s -X POST $BASE/api/auth/logout \
  -H "Content-Type: application/json" \
  -d "{\"refreshToken\":\"$REFRESH\"}"
# Expected: { "message": "Logged out successfully." }

# 7. Verify refresh is revoked
curl -s -X POST $BASE/api/auth/refresh \
  -H "Content-Type: application/json" \
  -d "{\"refreshToken\":\"$REFRESH\"}"
# Expected: 401
```

✅ All seven steps produce the expected results.
❌ If step 7 still returns a token, the logout did not delete the RefreshToken document — check the tokenHash computation.
