# What You Will Build

By the end of this chapter, EduFlow knows who every user is. Specifically:

| Endpoint | What it does |
|----------|-------------|
| POST /api/auth/register | Creates an account; sends verification OTP |
| POST /api/auth/verify-email | Confirms OTP; marks account verified |
| POST /api/auth/login | Returns access token + refresh token |
| POST /api/auth/refresh | Accepts refresh token; returns new access token |
| POST /api/auth/logout | Revokes the refresh token |
| GET /api/auth/me | Returns the authenticated user's profile |

This is the authentication layer that every other chapter depends on. You build it once, you build it right.

✅ After this chapter, you can register, verify, login, and get a protected response — all tested with curl.
❌ The email sending is stubbed (logged to console) until chapter 18 adds Resend. The flow works; real emails come later.
