# Why Authentication Matters

In 2012, LinkedIn suffered a data breach that exposed 6.5 million password hashes. The hashes were unsalted SHA-1 — a fast, general-purpose algorithm with no per-password randomness. Within days, the majority had been cracked using precomputed rainbow tables. Millions of users had their passwords exposed.

The engineers who built that system probably did not think hard about hashing. They stored something instead of plain text — that felt like enough. It was not.

The difference between SHA-1 and bcrypt, between no salt and a per-password salt, is the difference between a breach that destroys user trust and one that is survivable. That difference is twelve lines of code and one informed decision.

EduFlow stores student payment information and course progress. A compromised user account means a compromised student's history and potentially their payment method. Authentication is not a formality — it is foundational.

This chapter builds it right, from the start.
