# Wire Auth Routes Into the App

With the auth module folder and files in place, connect it to the Express app.

**In auth.routes.js** — add a minimum export so app.js can import it without errors:

```js
// src/modules/auth/auth.routes.js
import { Router } from 'express';
export const authRouter = Router();
// Individual routes are added in the next sub-chapters
```

**In app.js** — mount the router:

```js
// src/app.js
import { authRouter } from './modules/auth/auth.routes.js';

// After middleware, before error handler:
app.use('/api/auth', authRouter);
```

**Restart the server** and verify it still starts:

```
npm run dev
```

No errors expected. The router exists but has no routes yet.

**Verify the mount point:**

```
curl http://localhost:3000/api/auth/register
```

Expected: 404 with a message — the route does not exist yet, but the path is reachable.

✅ The server starts without errors after mounting the empty router.
❌ Do not add all routes at once before testing each one — add, verify, continue.
