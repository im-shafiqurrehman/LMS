# Mandatory Reads — Before You Build Auth

Read these before writing any auth code. The next sub-chapter builds directly on them.

**1. Password hashing (bcrypt / Argon2)**
Search "OWASP password storage cheat sheet". Understand why a salt prevents rainbow table attacks, and why iteration count (bcrypt cost factor) makes brute-force impractical. _Required: chapter 06.10 implements the User model with bcrypt._

**2. JWT explained**
Search "jwt.io introduction". The interactive decoder at jwt.io lets you paste a token and see its three parts. Understand what the signature protects and what it does not encrypt. _Required: you will sign and verify JWTs in 06.14._

**3. Sessions vs tokens: the stateless API design tradeoff**
Search "stateless authentication REST API best practices". Understand why session-based auth requires shared state (a problem for horizontal scaling) while JWT-based auth is stateless. _Required: you are making this architectural choice in this chapter._

These three reads together take about 30 minutes. They are not optional — they are the foundation for every decision you will make in this chapter.
