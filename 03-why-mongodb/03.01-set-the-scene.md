# Why MongoDB? — Set the Scene

Before you write a line of application code, you choose a database. This choice shapes every other decision that follows — the schema, the queries, the performance work, the migration strategy. Getting it wrong early means expensive rewrites later.

This chapter makes that choice deliberately. It explains why EduFlow uses MongoDB and Mongoose, what it gives up by not using a relational database, and how the data model maps to MongoDB's document model.

This is a **concept chapter** — there is no code to write. By the end, you should be able to explain the choice to someone who asks "why not PostgreSQL?" — and give a real answer, not "because the tutorial said so."
