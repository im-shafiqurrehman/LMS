# The LMS Data Model at a Glance

Here is the complete logical picture of EduFlow's data — the collections you will build and how they relate:

```
User
  _id, email, passwordHash (select:false), role (student|instructor|admin)
  emailVerifiedAt, verificationOtp (select:false), otpExpiresAt (select:false)
  timestamps

InstructorProfile
  _id, userId → User, bio, photoUrl, isApproved
  timestamps

Category
  _id, name, slug

Course
  _id, instructorId → User, categoryId → Category
  title, slug, description, coverImageUrl
  price, status (draft|pending|published|rejected)
  totalLessons, totalDuration
  timestamps

Lesson
  _id, courseId → Course
  title, videoUrl, durationSeconds, position
  timestamps

Enrollment
  _id, studentId → User, courseId → Course
  amountPaid (price snapshot), stripePaymentIntentId
  enrolledAt

ProgressRecord
  _id, enrollmentId → Enrollment, lessonId → Lesson
  completedAt

Certificate
  _id, enrollmentId → Enrollment
  pdfUrl, issuedAt

Question
  _id, courseId → Course, userId → User
  content
  timestamps

Answer
  _id, questionId → Question, instructorId → User
  content
  timestamps

RefreshToken
  _id, tokenHash, userId → User
  expiresAt
  timestamps
```

Notice amountPaid on Enrollment — this is a **price snapshot**. It records what the student actually paid at the time of enrollment. If the instructor raises the price next month, existing enrollments keep their original amount. This is a deliberate decision you will understand deeply in chapter 13.

You will define all these Mongoose schemas in chapter 05.
