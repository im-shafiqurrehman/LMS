# Before You Start — Prerequisites

This course assumes you can already build a basic CRUD API. It starts there and climbs. The table below is the floor the course stands on — not a list of everything the course teaches, but the minimum you need to be comfortable with before chapter 2.

If a row says "revise if rusty," spend an hour on it before starting. A shaky prerequisite slows every chapter that depends on it.

| You should be comfortable with | Why you need it | Status |
|---|---|---|
| Building a REST API with Node.js and Express (routes, middleware, `req`/`res`) | Every chapter builds on this foundation | **Must have** |
| MongoDB basics — collections, documents, basic queries | The data model and all queries assume this | **Must have** |
| `async/await` and Promises in JavaScript | Every database call, every API call, every file operation in this project is async | **Must have** |
| Everyday Git — commit, branch, `.gitignore`, pushing to a remote | You commit daily; secrets stay out of history | **Must have** |
| Reading and writing JSON | Every API endpoint speaks JSON | **Must have** |
| npm basics — installing packages, running scripts | You'll install packages in every chapter | **Must have** |
| Basic command line navigation (`cd`, `ls`, `mkdir`, reading error output) | You'll run commands constantly | **Must have** |
| What an HTTP request is — method, path, headers, body, status code | The course builds on this without re-teaching it | **Revise if rusty** |
| Environment variables — what they are and why secrets go there | Used from chapter 3 onward | **Revise if rusty** |

---

**Helpful but taught when it arrives:** TypeScript, Docker, Redis, Prisma, Stripe, Cloudinary, JWT. You do not need to know these before you start. Each is introduced from scratch, with the concept taught before the code.

**Not assumed:** React knowledge. The frontend chapters are guided carefully, but they are not the focus of this course. If your React is shaky, the chapters will carry you — the backend is where this course lives.

---

## Recommended warm-up (if anything above is rusty)

- **HTTP and REST:** Search "HTTP methods and status codes REST API" — the MDN article on HTTP is excellent.
- **async/await:** Search "JavaScript async await explained" — the official MDN guide covers the mental model clearly.
- **MongoDB:** Search "MongoDB tutorial for beginners" — the official MongoDB University introduction is worth an afternoon.
- **Git:** Search "Git handbook GitHub" — GitHub's own handbook is short and practical.
